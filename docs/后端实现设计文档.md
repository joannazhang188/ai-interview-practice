# AI 面试陪练 - 后端实现设计文档

## 一、技术架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   前端浏览器                          │
│  现有 Vanilla JS + CSS，改造加 fetch 调用后端 API     │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP
┌──────────────────────▼──────────────────────────────┐
│              FastAPI 后端 (Python)                    │
│  ├── 文件上传 + 解析（PDF/Word/图片 → 文本）           │
│  ├── 数据持久化（SQLite）                            │
│  ├── 简历/作品/面试记录 CRUD                         │
│  └── AI 分析 → 转发给 SOLO 会话获取结果              │
└─────────────────────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────┐
│        SOLO 会话（AI 实时分析）                       │
│  · 匹配度分析（5子维度评分 + 6张分析卡片）            │
│  · 面试问题生成（基于 JD + 简历）                    │
│  · 即时评分反馈（训练模式）                          │
│  · 报告生成（10维度评分）                            │
└─────────────────────────────────────────────────────┘
```

### 1.2 技术栈

- **前端**：现有 Vanilla JS + CSS（改造）
- **后端**：Python 3.9+ + FastAPI
- **数据库**：SQLite（零配置，MVP 最快）
- **文件解析**：PyMuPDF（PDF）、python-docx（Word）
- **AI 分析**：SOLO 会话（当前会话暂代，后续可切换 API）

### 1.3 AI 调用方式

后端 AI 分析不走 API 调用，而是通过**后端 → SOLO 会话**的方式：
- 后端准备好分析上下文（简历内容、JD 内容、对话历史等）
- 后端将分析请求写入一个共享文件或队列
- 前端/后端轮询结果，或通过轮询接口获取 SOLO 分析结果
- SOLO（我）实时读取请求、分析、写入结果

**备选简化方案（推荐 MVP）**：
- 后端将分析请求以结构化 JSON 存入数据库
- 前端轮询时，后端直接返回给前端
- 同时，前端在提交时触发一个 SOLO 分析请求
- SOLO 分析完成后，结果同样写入数据库
- 前端刷新即可看到结果

**当前 MVP 实现**：
- AI 分析请求写入数据库的 `pending_ai_requests` 表
- 前端轮询 `/api/ai/result/{request_id}` 获取结果
- SOLO 会话通过轮询或 webhook 触发（需另开一个 agent 监听）
- **更简单**：后端先返回"分析中"状态，前端展示 loading
- SOLO 读取数据库请求，分析后写入结果
- 前端通过轮询获取结果

### 3.2 表结构

```sql
-- 简历库
CREATE TABLE resumes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,
    version TEXT DEFAULT 'v1.0',
    is_default BOOLEAN DEFAULT 0,
    raw_text TEXT,              -- 解析后的纯文本
    parsed_data TEXT,            -- JSON: {skills: [], experience: [], education: []}
    parse_status TEXT DEFAULT 'pending',  -- pending/parsing/done/failed
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 作品库
CREATE TABLE portfolios (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_type TEXT,             -- doc/pic/xls/ppt/ps/pdf
    raw_text TEXT,
    parsed_data TEXT,            -- JSON: {summary, skills, highlights, suggested_roles}
    parse_status TEXT DEFAULT 'pending',
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- JD 资料库
CREATE TABLE jd_documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    company_name TEXT,
    job_title TEXT NOT NULL,
    raw_text TEXT,              -- 解析后的纯文本
    file_path TEXT,
    uploaded_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 匹配度分析实例
CREATE TABLE match_instances (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    jd_document_id INTEGER,
    resume_id INTEGER,
    portfolio_ids TEXT,         -- JSON array
    job_title TEXT,
    company_name TEXT,

    -- 分析结果
    total_score INTEGER,        -- 总分 0-100
    sub_scores TEXT,            -- JSON: {岗位经验: 16, 专业能力: 17, ...}
    analysis_cards TEXT,        -- JSON: 6张分析卡片完整内容
    threshold_result TEXT,      -- 'pass' / 'fail' / null

    status TEXT DEFAULT 'pending',  -- pending/analyzing/done/failed
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 面试实例
CREATE TABLE interviews (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    match_instance_id INTEGER,
    mode TEXT NOT NULL,         -- 'train' / 'sim'
    round_status TEXT DEFAULT 'hr',  -- hr/manager/boss/done

    -- 当前状态
    current_question_index INTEGER DEFAULT 0,
    total_questions INTEGER DEFAULT 0,
    status TEXT DEFAULT 'active',  -- active/paused/finished

    -- 评分结果（模拟模式结束后填充）
    total_score INTEGER,
    dimension_scores TEXT,      -- JSON: 10维度分数

    started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ended_at DATETIME
);

-- 面试消息记录
CREATE TABLE interview_messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    interview_id INTEGER,
    role TEXT NOT NULL,         -- 'ai' / 'user'
    content TEXT NOT NULL,
    question_index INTEGER,

    -- 训练模式即时反馈
    score INTEGER,              -- 0-100
    feedback TEXT,              -- JSON: {strengths, weaknesses, suggestions, optimized_answer}
    dimensions TEXT,            -- JSON: 10维度分数

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- AI 分析请求队列（SOLO 轮询）
CREATE TABLE ai_requests (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    request_type TEXT NOT NULL,  -- 'match_analysis' / 'question' / 'score' / 'report'
    context TEXT NOT NULL,        -- JSON: 完整的分析上下文
    result TEXT,                   -- JSON: SOLO 分析结果
    status TEXT DEFAULT 'pending', -- pending/processing/done/failed
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 四、API 设计

### 4.1 简历管理

```
POST   /api/resumes/upload
       上传简历文件（PDF/Word/图片）
       响应: {id, filename, parse_status: 'pending'}

GET    /api/resumes
       获取简历列表
       响应: [{id, filename, version, is_default, parse_status, skills, uploaded_at}]

GET    /api/resumes/{id}
       获取简历详情（含解析后的 raw_text, parsed_data）
       响应: {id, filename, raw_text, parsed_data, ...}

PUT    /api/resumes/{id}/default
       设为默认简历

DELETE /api/resumes/{id}
```

### 4.2 作品管理

```
POST   /api/portfolios/upload
GET    /api/portfolios
GET    /api/portfolios/{id}
DELETE /api/portfolios/{id}
```

### 4.3 JD 管理

```
POST   /api/jd/upload
       上传 JD 文件 + 填写公司/岗位信息
       响应: {id, job_title, parse_status}

GET    /api/jd
       获取 JD 列表
```

### 4.4 匹配度分析

```
POST   /api/match/analyze
       发起匹配度分析
       请求: {
         jd_document_id,
         resume_id,
         portfolio_ids: [1, 2, 3],
         job_title,
         company_name
       }
       响应: {request_id}  # 前端轮询 request_id 获取结果

GET    /api/ai/result/{request_id}
       轮询 AI 分析结果
       响应: {
         status: 'processing' | 'done' | 'failed',
         result: {total_score, sub_scores, analysis_cards, threshold_result} | null
       }

GET    /api/match/instances
       分析记录列表
       响应: [{id, company_name, job_title, total_score, status, created_at}]

GET    /api/match/instances/{id}
       单条分析详情
```

### 4.5 面试

```
POST   /api/interview/start
       开始面试
       请求: {match_instance_id, mode: 'train' | 'sim'}
       响应: {interview_id}

GET    /api/interview/{id}
       获取面试状态
       响应: {id, status, round_status, current_question_index, ...}

POST   /api/interview/{id}/answer
       提交回答
       请求: {content, question_index}
       响应: {request_id}  # 前端轮询获取评分结果（训练模式）或下一题（模拟模式）

GET    /api/interview/{id}/messages
       获取对话历史

POST   /api/interview/{id}/reverse-question
       用户反问
       请求: {content}
       响应: {request_id}

POST   /api/interview/{id}/finish
       结束面试（模拟模式调用，触发整场评分）
```

### 4.6 报告

```
GET    /api/interview/{id}/report
       获取面试报告
       响应: {request_id}  # 轮询获取

GET    /api/interviews
       面试历史列表
```

## 五、AI 分析流程

### 5.1 匹配度分析

```
1. 前端 POST /api/match/analyze
2. 后端：
   - 读取简历 raw_text + JD raw_text + 作品 raw_text
   - 构建分析上下文（JSON）
   - 写入 ai_requests 表，status='pending'
   - 返回 {request_id}
3. SOLO 轮询 ai_requests 表
4. SOLO 读取请求，分析：
   - 计算 5 子维度匹配度（关键词重叠、经验匹配度等）
   - 生成 6 张分析卡片（评分理由/优势/短板/重点表达/风险/建议）
   - 判断阈值（<65% 婉拒，>=65% 通过）
   - 写入 result 字段，status='done'
5. 前端轮询 GET /api/ai/result/{request_id}
6. 结果返回后前端渲染
```

### 5.2 面试问题生成

```
1. 前端 POST /api/interview/start
2. 后端：
   - 读取 match_instance 的 JD + 简历
   - 构建问题生成上下文
   - 写入 ai_requests 表
   - 返回 interview_id
3. SOLO 分析，生成第一个问题
4. 前端轮询获取第一个问题
```

### 5.3 评分反馈（训练模式）

```
1. 前端 POST /api/interview/{id}/answer
2. 后端写入 ai_requests（type='score'）
3. SOLO 分析回答，给出：
   - 单题得分
   - 10维度分数
   - 反馈（亮点/不足/优化建议/优化版回答）
   - 下一个问题
4. 前端轮询获取结果
```

### 6.3 轮询机制

```javascript
async function pollResult(requestId, callback) {
  const check = async () => {
    const res = await api.getResult(requestId);
    if (res.status === 'done') {
      callback(res.result);
    } else if (res.status === 'processing') {
      setTimeout(check, 2000);  // 2秒后再查
    }
  };
  check();
}
```

## 九、注意事项

1. **SOLO 会话依赖**：当前 AI 分析依赖 SOLO 会话。会话结束后需重启监听。
2. **文件大小限制**：上传文件限制 20MB。
3. **CORS 配置**：后端允许 localhost 前端访问。
4. **MVP 范围**：用户认证暂不做，单用户 MVP。
5. **轮询频率**：前端 2-3 秒轮询一次，避免过于频繁。
